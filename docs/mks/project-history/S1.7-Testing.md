# Sprint S1.7 — Testing

**Status:** 🟡 Em andamento

## Objetivo

Criar a infraestrutura de testes do Melody Sync e validar o comportamento dos módulos do Core.

## Implementado

### Infraestrutura

- README dos testes.
- Organização da pasta `tests`.
- Criação do `conftest.py`.
- Fixtures compartilhadas.

### Model: Song

- Testes para `filename`.
- Testes para `extension`.
- Testes para `directory`.
- Testes para `has_metadata`.
- Testes para `size_mb`.
- Testes para `size_gb`.
- Testes para `duration_minutes`.
- Testes para `is_lossless`.

## Decisões

- Cada teste valida um único comportamento.
- Fixtures representam cenários reais do domínio.
- Testes utilizam `pytest`.
- O desenvolvimento passa a seguir o ciclo:
    - implementar;
    - testar;
    - revisar;
    - documentar;
    - commit.

## Resultado

O modelo `Song` teve todos os seus comportamentos públicos validados por testes automatizados, tornando-se a referência para os próximos módulos.
