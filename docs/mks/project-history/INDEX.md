# Objetivo

Este diretório documenta toda a evolução do Melody Sync desde a primeira Sprint.

Cada Sprint representa uma entrega completa, contendo:

- objetivo;
- planejamento;
- implementação;
- revisão;
- testes;
- decisões arquiteturais;
- lições aprendidas.

O objetivo desta documentação é permitir que qualquer pessoa compreenda como o projeto evoluiu, quais decisões foram tomadas e por que elas foram adotadas.

# Metodologia

Cada Sprint segue o mesmo ciclo de desenvolvimento.

```text
Planejamento
      │
      ▼
Implementação
      │
      ▼
Code Review
      │
      ▼
Testes
      │
      ▼
Correções
      │
      ▼
Documentação
      │
      ▼
Commit
```

Uma Sprint somente é considerada concluída quando todas essas etapas forem finalizadas.
